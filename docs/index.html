The tracks

Seven tracks between now and 2026-08-09. Each has a single accountable owner; some items are jointly executed.
Track 1 · Product closure

Takanon — sellable core

OwnerTal

Every P0 gap that would embarrass in a live demo. Feature bar: a fresh customer with a real corpus can ask questions and get cited, honest answers on day one.

  • Original document viewer v2 — page-jump + passage highlight
    Click a citation → open PDF at the exact page, highlight the cited passage. v1 (open at page 1) already shipped; v2 lands the "wow" moment.
    Tal
  • "No confident answer" state — clear, distinct UI
    Refused already looks right. Uncertain answers need their own visual language so users don't confuse them with confident ones. Pairs with cite-or-refuse.
    Talia
  • Bulk import — integration platform TBD
    One-shot ingest of a tenant's existing archive tree — no manual reupload per file. Blocks the "we already have 200 documents somewhere" objection. Source platform (Google Drive, Dropbox, direct upload, etc.) to be decided based on where customers actually keep their archives.
    Tal
  • Empty states + first-run tour
    A brand-new tenant lands on Search with no docs uploaded → the UI guides them. Same on Documents / Lexicon / Amendments.
    Talia
  • Golden set — all P0 questions passing on El-Rom corpus
    Pass rate ≥ 90% on the P0 bucket. Foundation for regression testing post-launch.
    Tal

Track 4 · Legal + trust

The paperwork that lets us sign

OwnerTal

Israeli-law-appropriate. Cheap but blocking — no responsible customer signs without seeing these.

  • Terms of Service (Hebrew)
    Standard SaaS ToS adapted for Israeli entities. Draft first pass with Claude, review with a lawyer before go-live.
    Tal
  • Privacy policy (Hebrew)
    Israeli Privacy Protection Law + GDPR-adjacent. Explicit list of processors (Anthropic, Cohere, Render, Resend, Google).
    Tal
  • DPA template for enterprise contracts
    Data Processing Addendum, ready to attach to a customer contract. Doesn't have to be perfect — has to exist.
    Tal
  • Cookie disclosure banner
    Minimal, non-intrusive. Vanilla JS, no vendor. Gates Microsoft Clarity behind explicit consent; choice persisted in localStorage.
    Tal
  • Accessibility statement (Israeli IS 5568)
    Mandatory for Israeli services. See existing israeli-accessibility-compliance skill; drives concrete UX fixes as well.
    Tal + Talia
Track 5 · Reliability

Uptime floor for real customers

OwnerTal

Not "enterprise SRE" — just enough that we hear about breakage before customers do, and can respond during business hours.

  • Error alerting to a real inbox / phone
    Sentry-lite or Slack webhook from structlog. Every 500 in production notifies Tal within a minute.
    Tal
  • Uptime monitoring + public status page
    Better Uptime or equivalent. All three services (identity, takanon backend, meetings backend). Status page linked from landing.
    Tal
  • P50 / P95 latency dashboard
    Answer latency is the customer-visible metric. Alert on spikes — the debug-queue already covers wrong answers.
    Tal
  • Ingest failure alerts
    Bulk-import silently dropping files would be catastrophic post-go-live. Explicit alert on any ingestion exception.
    Tal
  • One-week soak on El-Rom, no P0 incidents
    Aug 2–9 as the freeze week: no new features, only fixes. If El-Rom has a bad week, we don't go live.
    Tal

How customers reach us when things break

The in-app "report an issue" button is the seed. Wrap it in real process so customers feel heard.

    Shipped: 🚩 button emails Tal with full session context. Basis for the wider support surface.
    Tal
  • Public FAQ / help page
    "How does citing work?" · "What if the answer is wrong?" · "How do I add users?" Kill the top 10 support tickets before they arrive.

Track 7 · Design

UX unification across the suite

OwnerTalia

Two products, one Klaser. Anything a customer sees on both surfaces must feel like the same company built them — colors, spacing, buttons, empty states, error copy.

  • Design token audit — colors, type, spacing, radii
    Extract the shared system from Takanon's Tailwind config; port to Meetings. Ink, accent, coral, line — one truth.
    Talia
  • Shared component patterns — buttons, modals, tables, empty states
    Not a component library — no time for that. A single-page spec: "this is what a button looks like across Klaser," reproduced in both codebases.
    Talia
  • Hebrew RTL polish audit — both products
    Icons, arrows, list markers, form validation messages. RTL bugs are the fastest way to signal "this app wasn't built for us."
    Talia
  • Loading + error + empty states — consistent voice
    One Hebrew tone across both products (warm, plainspoken, not corporate). Pairs with the empty-state work in Tracks 1 & 2.
    Talia
  • Micro-interactions — the small delights
    The mesh animation on the landing was a first taste. Talia decides where in the products a similar move earns its keep.
    Talia
